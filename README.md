# Another Box

Минималистичный Windows GUI для запуска JSON-подписок sing-box.

## Запуск

Требуется Python 3.12 или новее.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[test]"
```

Поместите Windows-бинарник в `bin\sing-box.exe`, затем запустите:

```powershell
python -m another_box
```

Профили сохраняются в `%APPDATA%\AnotherBox\profiles`. При закрытии окна
приложение продолжает работу в системном трее. Для полной остановки используйте
пункт «Выход» в меню трея.

TUN-профили требуют запуска Python или терминала от имени администратора.

Ошибки самого приложения и фоновых операций записываются в
`%APPDATA%\AnotherBox\logs\application.log`. Журнал можно открыть кнопкой
«Журнал» в главном окне или через меню системного трея.

## Требования к подписке

Из JSON-подписки в итоговый `config.json` копируются только следующие секции:

- `outbounds`
- `endpoints`
- `route`
- `dns`

Секция `inbounds` всегда формируется из настроек профиля в приложении.
Остальные секции подписки игнорируются и не передаются sing-box. Полный
исходный JSON при этом сохраняется локально как `source.json` для диагностики.

Подписка должна содержать хотя бы один outbound или endpoint. Один из них
должен иметь фиксированный tag `PROXY`. Приложение проверяет его наличие,
но не добавляет и не переименовывает элементы подписки.

Пример outbound:

```json
{
  "outbounds": [
    {
      "type": "selector",
      "tag": "PROXY",
      "outbounds": ["proxy-node"]
    }
  ]
}
```

Правила маршрутизации, которые должны отправлять трафик в этот outbound, также
должны ссылаться на `PROXY`, например:

```json
{
  "action": "route",
  "outbound": "PROXY"
}
```

Tag управляемого inbound задается отдельно в интерфейсе. По умолчанию
используются `mixed-in` для mixed и `tun-in` для TUN.
