# Another Box

Минималистичный Windows GUI для запуска JSON-подписок sing-box.

## Запуск

Требуется Python 3.12 или новее.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[test]"
```

Поместите Windows-бинарник в `bin\sing-box.exe`, затем запустите:

```powershell
python -m another_box
```

Профили сохраняются в `%APPDATA%\AnotherBox\profiles`. При закрытии окна
приложение продолжает работу в системном трее. Для полной остановки используйте
пункт «Выход» в меню трея.

TUN-профили требуют запуска Python или терминала от имени администратора.

Ошибки самого приложения и фоновых операций записываются в
`%APPDATA%\AnotherBox\logs\application.log`. Журнал можно открыть кнопкой
«Журнал» в главном окне или через меню системного трея.

Ошибки обновления и запуска не открывают модальные окна и не блокируют
интерфейс. Состояние отображается непосредственно в карточке профиля; полный
текст ошибки доступен при наведении и в журнале приложения.

Для каждого профиля можно включить автообновление подписки и задать интервал в
минутах. Минимальный интервал — 30 минут. Планировщик продолжает работать, когда
окно свернуто в системный трей. Активный профиль после обновления продолжает
работать со старой конфигурацией до ручного перезапуска.

Для каждого профиля отдельно настраивается логирование sing-box: включение,
уровень (`trace`, `debug`, `info`, `warn`, `error`, `fatal`, `panic`) и
добавление временных меток. Секция `log` формируется приложением и не берется
из подписки. Вывод sing-box сохраняется в журнале соответствующего профиля.
Кнопка с иконкой папки в карточке профиля открывает его каталог с файлом
`process.log`.

## Требования к подписке

Из JSON-подписки в итоговый `config.json` копируются только следующие секции:

- `outbounds`
- `endpoints`
- `route`
- `dns`

Секция `inbounds` всегда формируется из настроек профиля в приложении.
Остальные секции подписки игнорируются и не передаются sing-box. Полный
исходный JSON при этом сохраняется локально как `source.json` для диагностики.
Поддерживается JSONC-синтаксис sing-box: комментарии и завершающие запятые.

Подписка должна содержать outbound с фиксированным tag `PROXY`. Приложение
проверяет его наличие, но не добавляет и не переименовывает outbounds.

Пример outbound:

```json
{
  "outbounds": [
    {
      "type": "selector",
      "tag": "PROXY",
      "outbounds": ["proxy-node"]
    }
  ]
}
```

Правила маршрутизации, которые должны отправлять трафик в этот outbound, также
должны ссылаться на `PROXY`, например:

```json
{
  "action": "route",
  "outbound": "PROXY"
}
```

Управляемый inbound всегда создается с фиксированным tag `IN`. Изменить этот
tag через интерфейс нельзя. Приложение не изменяет правила маршрутизации
подписки, поэтому правила с условием по inbound должны заранее ссылаться на
`IN`, например:

```json
{
  "inbound": ["IN"],
  "action": "route",
  "outbound": "PROXY"
}
```

## Создание и запуск профиля

При добавлении профиля приложение скачивает подписку, формирует локальный
`config.json` и сохраняет его без запуска sing-box и без выполнения
`sing-box check`.

Если подписку не удалось скачать или обработать, профиль все равно остается в
списке. Ошибка сохраняется в метаданных и журнале; загрузку можно повторить
командой «Обновить подписку».

Проверка конфигурации через `sing-box check` выполняется после нажатия
«Запустить» в главном окне. Только после успешной проверки запускается процесс
sing-box. Наличие outbound с tag `PROXY` также проверяется только перед ручным
запуском. Последующие обновления существующего профиля по-прежнему проверяются
до замены последней рабочей конфигурации.
